# Спецификация формата конфигураций

## Общая структура

Конфигурация — JSON-файл со следующими полями:

```json
{
    "j_x": -48,
    "c_x": -160,
    "cd_len": 65,
    "de_len": 20,
    "fg_len": 20,
    "gh_len": 70,
    "hi_len": 43.5,
    "jk_len": 8.5,
    "hcor": 80,
    "image": null
}
```

Опциональное поле `_comment` может присутствовать в системных пресетах для
документирования соответствия полей параметрам H.

---

## Описание полей

### j_x

| Свойство | Значение |
|----------|---------|
| Имя | `j_x` |
| Тип | `number` (float) |
| Назначение | Координата точки J по оси X (параметр H1) |
| Допустимые значения | Отрицательные числа (левая сторона контура) |
| Влияние на расчёт | Определяет горизонтальное положение точек J и K |
| Формула | `K_X = j_x` |
| Пример | `-48` |

### c_x

| Свойство | Значение |
|----------|---------|
| Имя | `c_x` |
| Тип | `number` (float) |
| Назначение | Координата точки C по оси X (параметр H2) |
| Допустимые значения | Отрицательные числа (левая сторона контура) |
| Влияние на расчёт | Определяет горизонтальное положение точек C и D |
| Формула | `D_X = c_x` |
| Пример | `-160` |

### cd_len

| Свойство | Значение |
|----------|---------|
| Имя | `cd_len` |
| Тип | `number` (float) |
| Назначение | Длина сегмента CD (параметр H3) |
| Допустимые значения | ≥ 0 |
| Влияние на расчёт | Вертикальное расстояние между точками C и D |
| Формула (REV=False) | `D_Y = C_Y - cd_len` |
| Формула (REV=True) | `D_Y = C_Y + cd_len` |
| Пример | `65` |

### de_len

| Свойство | Значение |
|----------|---------|
| Имя | `de_len` |
| Тип | `number` (float) |
| Назначение | Длина сегмента DE (параметр H4) |
| Допустимые значения | ≥ 0 |
| Влияние на расчёт | Расстояние от D до E вдоль направления alpha |
| Формула | `E = D ± de_len * (cos(alpha), sin(alpha))` (зависит от REV) |
| Пример | `20` |

### fg_len

| Свойство | Значение |
|----------|---------|
| Имя | `fg_len` |
| Тип | `number` (float) |
| Назначение | Длина сегмента FG (параметр H5) |
| Допустимые значения | ≥ 0 |
| Влияние на расчёт | Расстояние от F до G вдоль направления alpha |
| Формула | `G = F - fg_len * (cos(alpha), -sin(alpha))` |
| Пример | `20` |

### gh_len

| Свойство | Значение |
|----------|---------|
| Имя | `gh_len` |
| Тип | `number` (float) |
| Назначение | Длина сегмента GH (параметр H6) |
| Допустимые значения | ≥ 0 |
| Влияние на расчёт | Вертикальное расстояние между точками G и H |
| Формула | `H_Y = G_Y + gh_len` |
| Пример | `70` |

### hi_len

| Свойство | Значение |
|----------|---------|
| Имя | `hi_len` |
| Тип | `number` (float) |
| Назначение | Длина сегмента HI (параметр H7) |
| Допустимые значения | ≥ 0 |
| Влияние на расчёт | Горизонтальное расстояние между точками H и I |
| Формула | `I_X = H_X + hi_len` |
| Пример | `43.5` |

### jk_len

| Свойство | Значение |
|----------|---------|
| Имя | `jk_len` |
| Тип | `number` (float) |
| Назначение | Длина сегмента JK (параметр H8) |
| Допустимые значения | ≥ 0 |
| Влияние на расчёт | Вертикальное расстояние между точками J и K |
| Формула | `J_Y = K_Y - jk_len` |
| Пример | `8.5` |

### hcor

| Свойство | Значение |
|----------|---------|
| Имя | `hcor` |
| Тип | `number` (float) |
| Назначение | Коррекция для расчёта n4 |
| Допустимые значения | ≥ 0 |
| Влияние на расчёт | При заданном n4: `внутренний_n4 = введённый_n4 + hcor` |
| Формула | `self.n4 = n4 + self.HCOR` (если n4 не None) |
| Пример | `80` |

### image

| Свойство | Значение |
|----------|---------|
| Имя | `image` |
| Тип | `string` или `null` |
| Назначение | Имя изображения конфигурации (для отображения в интерфейсе) |
| Допустимые значения | Имя файла или `null` |
| Влияние на расчёт | Не влияет |
| Пример | `null` |

---

## Соответствие полей параметрам H

| Поле JSON | Параметр H | Описание |
|-----------|-----------|---------|
| `j_x` | H1 | Координата J по X |
| `c_x` | H2 | Координата C по X |
| `cd_len` | H3 | Длина CD |
| `de_len` | H4 | Длина DE |
| `fg_len` | H5 | Длина FG |
| `gh_len` | H6 | Длина GH |
| `hi_len` | H7 | Длина HI |
| `jk_len` | H8 | Длина JK |

---

## Пример: системный пресет «Премьер комплект»

```json
{
    "_comment": {
        "c_x": "H2",
        "cd_len": "H3",
        "de_len": "H4",
        "fg_len": "H5",
        "gh_len": "H6",
        "hi_len": "H7",
        "j_x": "H1",
        "jk_len": "H8"
    },
    "c_x": -160,
    "cd_len": 65,
    "de_len": 20,
    "fg_len": 20,
    "gh_len": 70,
    "hi_len": 43.5,
    "j_x": -48,
    "jk_len": 8.5,
    "hcor": 80,
    "image": null
}
```

## Формат входного файла для CLI

Для CLI используется формат с секциями `config` и `input_data`:

```json
{
    "config": {
        "j_x": -48,
        "c_x": -160,
        "cd_len": 65,
        "de_len": 20,
        "fg_len": 20,
        "gh_len": 70,
        "hi_len": 43.5,
        "jk_len": 8.5,
        "hcor": 80
    },
    "input_data": {
        "n1": 100,
        "n2": null,
        "n4": null,
        "angle_EF": 5
    }
}
```

Поля `input_data`:
* `n1` — расстояние до точки B (число или `null`)
* `n2` — расстояние E–F (число или `null`)
* `n4` — расстояние I–K с коррекцией (число или `null`)
* `angle_EF` — угол наклона 0°–10° (число или `null`)

**Ровно три из четырёх параметров должны быть заданы (не null).**
