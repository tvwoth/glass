**Проверка установки:**
```bash
python --version

git --version
```

## Способ 1: Запуск через Docker (рекомендуется)

Это самый простой и надёжный способ. Docker автоматически настроит всё необходимое окружение.

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/tvwoth/glass.git
cd glass
```

Или, если вы уже скачали проект в виде ZIP-архива:
```bash
# Распакуйте архив и перейдите в папку
cd contour-app
```

### Шаг 2: Создание файла конфигурации

Создайте файл `.env` на основе примера:

**Windows (PowerShell):**
```powershell
Copy-Item .env.example .env
```

**Windows (CMD):**
```cmd
copy .env.example .env
```

**Linux/macOS:**
```bash
cp .env.example .env
```

### Шаг 3: Настройка переменных (опционально)

Откройте файл `.env` в текстовом редакторе и при необходимости измените параметры:

```env
# Внутренний порт приложения (обычно не меняется)
APP_PORT=5000

# Внешний порт (порт, на котором будет доступно приложение в браузере)
# 80 = http://localhost
# 8080 = http://localhost:8080
HOST_HTTP_PORT=80

# Пароль администратора для сохранения конфигураций
CONFIG_ADMIN_PASSWORD=admin

# Секретный ключ для сессий (оставьте пустым, сгенерируется автоматически)
# SECRET_KEY=
```

**Важно:** Если вы хотите использовать порт отличный от 80 (например, 8080), измените строку:
```env
HOST_HTTP_PORT=8080
```

### Шаг 4: Запуск контейнеров

```bash
docker compose up -d --build
```

**Что происходит:**
- `docker compose up` — запускает сервисы
- `-d` — запускает в фоновом режиме (detached)
- `--build` — собирает образ приложения из Dockerfile

**Первый запуск может занять 2-5 минут**, так как Docker скачивает базовый образ Python и устанавливает зависимости.

### Шаг 5: Проверка статуса

```bash
docker compose ps
```

Вы должны увидеть два запущенных контейнера:
- `glass` (приложение)
- `glass-nginx` (веб-сервер)

Оба должны иметь статус `Up` или `healthy`.

### Шаг 6: Открытие приложения

Откройте браузер и перейдите по адресу:

```
http://localhost
```

Или, если вы изменили порт:
```
http://localhost:8080
```

**Готово!** Приложение запущено.

## Способ 2: Ручной запуск через Python

Этот способ подходит для разработки или если Docker недоступен.

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/tvwoth/glass.git
cd glass
```

### Шаг 2: Создание виртуального окружения

**Windows (PowerShell):**
```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

**Windows (CMD):**
```cmd
python -m venv .venv
.venv\Scripts\activate.bat
```

**Linux/macOS:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

**Что происходит:**
- `python -m venv .venv` — создаёт изолированное окружение Python в папке `.venv`
- Активация окружения позволяет устанавливать пакеты без влияния на системный Python

После активации вы должны видеть префикс `(.venv)` в командной строке.

### Шаг 3: Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

**Что устанавливается:**
- Flask — веб-фреймворк
- gunicorn — WSGI-сервер для production
- matplotlib — построение графиков
- reportlab — генерация PDF
- pytest — тестирование
- и другие зависимости

### Шаг 4: Настройка переменных окружения

**Windows (PowerShell):**
```powershell
$env:CONFIG_ADMIN_PASSWORD="admin"
$env:SECRET_KEY="your-secret-key-here"
```

**Windows (CMD):**
```cmd
set CONFIG_ADMIN_PASSWORD=admin
set SECRET_KEY=your-secret-key-here
```

**Linux/macOS:**
```bash
export CONFIG_ADMIN_PASSWORD=admin
export SECRET_KEY="your-secret-key-here"
```

**Или создайте файл `.env`** (как в способе с Docker) и используйте его.

### Шаг 5: Запуск приложения

```bash
python -m app
```

Приложение запустится на `http://localhost:5000`.

**Для остановки** нажмите `Ctrl+C` в терминале.

Или для production с gunicorn:
```bash
gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app()"
```

**Параметры:**
- `-w 4` — 4 рабочих процесса
- `-b 0.0.0.0:5000` — слушать на всех интерфейсах, порт 5000

### Шаг 6: Открытие приложения

Откройте браузер и перейдите по адресу:

```
http://localhost:5000
```

**Готово!** Приложение запущено.

## Проверка работоспособности

### 1. Проверка через браузер

Откройте `http://localhost` (Docker) или `http://localhost:5000` (ручной запуск).

Вы должны увидеть главную страницу с:
- Блоком «Конфигурация»
- Полями ввода параметров (n1, n2, n4, angle_EF)
- Кнопкой «Рассчитать»

### 2. Проверка через curl

```bash
curl -s http://localhost:5000/ | head -10
```

Должен вернуться HTML-код страницы.

### 3. Проверка API

```bash
curl -s -X POST http://localhost:5000/api/calculate \
  -H "Content-Type: application/json" \
  -d "{\"config\":{\"j_x\":-48,\"c_x\":-160,\"cd_len\":65,\"de_len\":20,\"fg_len\":20,\"gh_len\":70,\"hi_len\":43.5,\"jk_len\":8.5,\"hcor\":80},\"input_data\":{\"n1\":100,\"n2\":50,\"n4\":null,\"angle_EF\":5}}"
```

Должен вернуться JSON с координатами точек.

### 4. Запуск тестов

```bash
# Активируйте виртуальное окружение (только для ручного запуска)
# Windows: .venv\Scripts\activate
# Linux/macOS: source .venv/bin/activate

python -m pytest tests/ -v
```

Все 50 тестов должны пройти успешно.

## Основные команды CLI

CLI (командная строка) позволяет работать с проектом без веб-интерфейса.

### Активация окружения (только для ручного запуска)

**Windows:**
```cmd
.venv\Scripts\activate
```

**Linux/macOS:**
```bash
source .venv/bin/activate
```

### Список доступных конфигураций

```bash
python -m app.cli config-list
```

Вывод:
```
Системные конфигурации:
  - Премьер комплект YP01YP02YP03G01
  - Синдикат КЗ471КЗ472КЗ473

Пользовательские конфигурации:
  (пусто)
```

### Выполнение расчёта

Создайте файл `input.json`:
```json
{
    "config": {
        "j_x": -48,
        "c_x": -160,
        "cd_len": 65,
        "de_len": 20,
        "fg_len": 20,
        "gh_len": 70,
        "hi_len": 43.5,
        "jk_len": 8.5,
        "hcor": 80
    },
    "input_data": {
        "n1": 100,
        "n2": 50,
        "n4": null,
        "angle_EF": 5
    }
}
```

Запустите расчёт:
```bash
python -m app.cli calculate input.json
```

Результат сохранится в `input.json.result.json`.

### Построение графика

```bash
python -m app.cli render input.json output.png
```

### Экспорт в PDF

```bash
python -m app.cli export-pdf input.json output.pdf
```

### Создание пользовательской конфигурации

```bash
python -m app.cli config-edit MyConfig j_x=-48 c_x=-160 cd_len=65 de_len=20
```

### Просмотр справки

```bash
python -m app.cli --help
```

## Частые проблемы и решения

### Проблема 1: `docker: command not found`

**Причина:** Docker не установлен или не добавлен в PATH.

**Решение:**
1. Скачайте Docker Desktop с https://www.docker.com/products/docker-desktop
2. Установите и перезагрузите компьютер
3. Запустите Docker Desktop
4. Повторите команду

### Проблема 2: `Permission denied` при запуске Docker

**Причина:** Недостаточно прав для доступа к Docker.

**Решение (Linux):**
```bash
sudo usermod -aG docker $USER
# Перелогиньтесь в систему
```

### Проблема 3: Порт 80 уже занят

**Симптом:** Ошибка `Bind for 0.0.0.0:80 failed: port is already allocated`

**Решение:**
1. Откройте `.env`
2. Измените `HOST_HTTP_PORT=80` на `HOST_HTTP_PORT=8080`
3. Перезапустите: `docker compose up -d --build`
4. Откройте `http://localhost:8080`

### Проблема 4: `python: command not found`

**Причина:** Python не установлен или не добавлен в PATH.

**Решение:**
1. Скачайте Python с https://www.python.org/downloads/
2. При установке отметьте «Add Python to PATH»
3. Перезапустите терминал
4. Проверьте: `python --version`

### Проблема 5: Ошибка при установке зависимостей

**Симптом:** `pip install -r requirements.txt` завершается с ошибкой.

**Решение:**
```bash
# Обновите pip
python -m pip install --upgrade pip

# Установите зависимости по одной
pip install flask
pip install gunicorn
pip install matplotlib
pip install reportlab
# ... и так далее
```

### Проблема 6: Приложение не открывается в браузере

**Проверьте:**

1. **Контейнеры запущены?**
   ```bash
   docker compose ps
   ```
   Оба контейнера должны быть `Up`.

2. **Порт правильный?**
   - Docker: `http://localhost` (или порт из `.env`)
   - Ручной запуск: `http://localhost:5000`

3. **Фаерволл не блокирует?**
   ```bash
   # Windows: проверьте, разрешён ли доступ для Python/Docker в фаерволле
   ```

4. **Логи приложения:**
   ```bash
   docker compose logs -f app
   ```

### Проблема 7: `ModuleNotFoundError: No module named 'app'`

**Причина:** Вы находитесь не в корневой папке проекта.

**Решение:**
```bash
# Перейдите в папку с проектом
cd contour-app

# Или укажите полный путь
python -m contour-app.app
```

### Проблема 8: Ошибка при активации виртуального окружения (Windows PowerShell)

**Симптом:** `Execution Policy Error`

**Решение:**
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
# Введите Y и нажмите Enter
.venv\Scripts\Activate.ps1
```
