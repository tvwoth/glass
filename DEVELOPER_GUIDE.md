Проект использует следующие скрипты для управления:

| Скрипт | Команда | Описание |
|--------|---------|----------|

## Защита конфигураций (административный пароль)

Пароль администратора используется для защиты операций с конфигурациями.
Реализация находится в `app/__init__.py`.

### Хеширование пароля

Пароль поддерживает два формата:

1. **werkzeug.security hash** — если пароль начинается с `pbkdf2:sha...`
2. **Открытый текст** — для обратной совместимости с существующими конфигурациями

Функция проверки пароля:

```python
def _check_password(plain_password: str, stored_password: str) -> bool:
    if stored_password.startswith('pbkdf2:sha'):
        return werkzeug.security.check_password_hash(stored_password, plain_password)
    return plain_password == stored_password
```

### Где требуется пароль

**Требуется:**
* Создание конфигурации
* Редактирование конфигурации
* Удаление конфигурации
* Импорт конфигурации
* Переименование конфигурации

**Не требуется:**
* Расчёты
* Просмотр результатов
* Экспорт PDF, CSV, JSON
* Работа с графиками
* Выбор конфигурации

## Улучшения UX (веб-интерфейс)

### localStorage для сохранения данных

При изменении любого поля формы, данные автоматически сохраняются в `localStorage`.
При обновлении страницы данные восстанавливаются.

Ключи:
* `contour_form_data` — данные формы расчёта и H-параметров
* `contour_hparams_open` — состояние панели "Свои настройки"

### Состояние панели настроек

Панель "Свои настройки" остаётся открытой после:
* Расчёта
* Выбора конфигурации
* Обновления параметров

Сворачивание — только по явному действию пользователя.

## Работа со знаками

### Конвенция

* Координаты `j_x` и `c_x` всегда отрицательные (левая сторона контура).
* В web-интерфейсе (`app/configs/helpers.py`, функция `parse_h_params`) к введённым
  значениям применяется операция `-abs()`.
* В API и CLI значения передаются как есть — отрицательные.

### Запреты

* Операции `+` и `-` являются частью математической модели. Их упрощение запрещено.
* Изменение знаков без анализа влияния на все точки запрещено.
* Любые преобразования вида `abs()`, `-abs()` должны быть зафиксированы в документации.

---

## Работа с углами

### Конвенция

* `angle_EF` — угол наклона сегмента EF (0°–10°).
* `angle_D = 180 + angle_EF` — угол D вычисляется из angle_EF.
* `alpha = 270 - angle_D` — вспомогательный угол для тригонометрических вычислений.
* Углы задаются в градусах, в формулах конвертируются в радианы через `math.radians()`.

### Запреты

* Направление отсчёта углов изменять запрещено.
* Нормализацию углов изменять запрещено.
* Порядок вычислений `angle_D → alpha` изменять запрещено.

Формат JSON (подробнее в [CONFIG_FORMAT.md](CONFIG_FORMAT.md)):

Расположение: `app/user_configs/`

* Веб-интерфейс (режим администратора)

## CLI

### Доступные команды

#### `calculate` — Выполнить расчёт

```bash
python -m app.cli calculate <config.json>
```

* **Аргументы:** путь к JSON-файлу конфигурации
* **Действие:** выполняет расчёт и выводит результат в консоль
* **Побочный эффект:** автоматически сохраняет `<config>.result.json`

#### `render` — Построить график

```bash
python -m app.cli render <config.json> [output.png]
```

#### `export-pdf` — Экспорт в PDF

```bash
python -m app.cli export-pdf <config.json> [output.pdf]
```

#### `config-list` — Список конфигураций

```bash
python -m app.cli config-list
```

#### `config-edit` — Создать/изменить конфигурацию

```bash
python -m app.cli config-edit <name> [key=value ...]
```
