# REST API — Contour App

## Базовый URL

```
http://localhost:5000
```

В Docker-режиме через Nginx:
```
http://SERVER_IP
```

---

## Маршруты

### POST /api/calculate

Выполняет расчёт геометрии контура стеклопакета.

#### Параметры запроса

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|---------|
| `config` | object | Да | Конфигурация параметров H |
| `config.j_x` | number | Да | Координата J по X (H1) |
| `config.c_x` | number | Да | Координата C по X (H2) |
| `config.cd_len` | number | Да | Длина CD (H3) |
| `config.de_len` | number | Да | Длина DE (H4) |
| `config.fg_len` | number | Да | Длина FG (H5) |
| `config.gh_len` | number | Да | Длина GH (H6) |
| `config.hi_len` | number | Да | Длина HI (H7) |
| `config.jk_len` | number | Да | Длина JK (H8) |
| `config.hcor` | number | Да | Коррекция n4 |
| `config.rev` | boolean | Нет | Реверс направления (по умолчанию `false`) |
| `input_data` | object | Да | Входные параметры расчёта |
| `input_data.n1` | number\|null | Условно | Расстояние до B |
| `input_data.n2` | number\|null | Условно | Расстояние E–F |
| `input_data.n4` | number\|null | Условно | Расстояние I–K (с коррекцией) |
| `input_data.angle_EF` | number\|null | Условно | Угол наклона (0–10°) |

**Ровно три из четырёх полей `input_data` должны быть числами (не null).**

#### Пример запроса

```bash
curl -X POST http://localhost:5000/api/calculate \
  -H "Content-Type: application/json" \
  -d '{
    "config": {
        "j_x": -48,
        "c_x": -160,
        "cd_len": 65,
        "de_len": 20,
        "fg_len": 20,
        "gh_len": 70,
        "hi_len": 43.5,
        "jk_len": 8.5,
        "hcor": 80,
        "rev": false
    },
    "input_data": {
        "n1": 100,
        "n2": 50,
        "n4": null,
        "angle_EF": 5
    }
}'
```

#### Ответ (успех, 200)

```json
{
    "n1": 100,
    "n2": 50,
    "n4": 120.15,
    "n3": 40.15,
    "angle_EF": 5,
    "angle_D": 185,
    "alpha": 85,
    "points": [
        [0.0, 0.0],
        [0.0, -100.0],
        [-160.0, -100.0],
        [-160.0, -165.0],
        [-140.17, -184.92],
        [-141.74, -135.08],
        [-159.83, -134.92],
        [-159.83, -64.92],
        [-116.33, -64.92],
        [-48.0, -8.5],
        [-48.0, 0.0]
    ]
}
```

| Поле | Тип | Описание |
|------|-----|---------|
| `n1` | number | Расстояние до B |
| `n2` | number | Расстояние E–F |
| `n4` | number | Расстояние I–K (с коррекцией hcor) |
| `n3` | number | Расстояние I–J (= n4 − hcor) |
| `angle_EF` | number | Угол наклона EF (градусы) |
| `angle_D` | number | Угол D (= 180 + angle_EF) |
| `alpha` | number | Вспомогательный угол (= 270 − angle_D) |
| `points` | array | Список из 11 массивов [x, y]: A, B, C, D, E, F, G, H, I, J, K |

#### Ответ (ошибка, 400)

```json
{
    "error": "Необходимо ввести ровно три параметра"
}
```

| Код | Описание |
|-----|---------|
| 400 | Неверные входные данные (валидация) |
| 500 | Внутренняя ошибка сервера |

#### Возможные ошибки

| Сообщение | Причина |
|-----------|---------|
| `No JSON data provided` | Тело запроса не содержит JSON |
| `Необходимо ввести ровно три параметра` | Меньше или больше 3 параметров |
| `Угол наклона должен быть в диапазоне 0°–10°` | angle_EF вне диапазона |
| `Длина ... должна быть положительной` | n1, n2 или n4 ≤ 0 |
| `Выберите конфигурацию или введите значения H` | Все H-параметры = 0 |
| `Не удалось найти длину n1/n2` | Бинарный поиск не сошёлся |
| `Не удалось найти угол D` | Поиск угла не сошёлся |

## Веб-интерфейс (Flask routes)

| Метод | Маршрут | Описание |
|-------|---------|---------|
| GET | `/` | Главная страница |
| POST | `/` | Выполнение расчёта (форма) |
| POST | `/config_admin/login` | Вход в режим администратора |
| POST | `/config_admin/logout` | Выход из режима администратора |
| POST | `/save_config` | Сохранение конфигурации (требует авторизации) |
| POST | `/delete_config` | Удаление конфигурации (требует авторизации) |
| POST | `/rename_config` | Переименование конфигурации (требует авторизации) |
